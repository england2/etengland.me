## Note
This page gives a broad overview of my Shellbin project, which I made to experiment with end-to-end CI/CD for microservice applications in Kubernetes.

You may be more interested reading a guide I wrote [that you can read here](/k8s-easy-ci-cd-guide/) which shows the complete technical to set a simple CI/CD for microservice applications in Kubernetes.

## Overview

There are two main aspects of this project:
1. Creating the micoservice software
2. Programatically deploying the software onto a Kubernetes cluster.


## Microservices?

_Some closing thoughts about this project._

There's a youtube video titled ["microservices are technical debt"](https://www.youtube.com/watch?v=LcJKxPXYudE).
To quickly summarize, a staff engineer at Door Dash says that:
- Microservices can be useful when multiple teams need to work independently on different parts of a system.
- In the long-term microservices can create developer pain and often aren't as decoupled as they are supposed to be.

<br>

Because I am an individual and not a team, it was diffuclt to appreciate the benefits of microservices while working on this project. 

<br>

Throughout the project, I was navigating an unecesarry web of complexity (and was patting myself on the pack for succesfully doing so, rather than mitigating the complexity in the first place!)

Although a monolith design would have been better for the feature-set of Shellbin, I am still glad that I chose to implement the program as I did.

The primary goal of the project was to learn a bit about Kubernetes, CI/CD, and microservice architecture, which I definitely diddid
